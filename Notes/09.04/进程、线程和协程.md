# 进程、线程和协程

进程、线程和协程是实现程序并发执行的三个层级。

**进程**是系统资源分配的最小单位（独立内存空间），

**线程**是CPU调度的最小单位（共享进程资源），

**协程**是用户态的轻量级线程（协作式任务），通过非抢占式切换来实现极高效率的IO密集型并发任务。

#### 1. 进程 (Process) - “工厂”

**定义**：一个正在运行的程序实例，是系统分配资源（内存、文件句柄等）的最小单位。

**特点**：每个进程拥有独立的地址空间。一个进程崩溃不会影响其他进程，但进程间通信（IPC）复杂且切换开销大。

**适用场景**：CPU密集型任务（如视频编码、复杂计算），需要高度并行且相互独立的任务。

#### 2. 线程 (Thread) - “工厂的工人”

**定义**：进程的一个执行路径，是CPU调度和分派的基本单位。

**特点**：线程是轻量级的进程，多个线程共享同一进程的内存和资源（代码、数据、文件），但拥有独立的寄存器和栈。线程切换较快，但存在线程安全问题。

**适用场景**：IO密集型任务（如网络请求、数据库读写）、需要响应多任务的程序。

#### 3. 协程 (Coroutine) - “工人分时作业”

**定义**：一种用户态的轻量级线程，也称微线程或纤程。

**特点**：不由操作系统调度，而是由程序本身（用户态）调度。协程切换开销极小（仅保存上下文），是非抢占式的（需要显式挂起/恢复），适合处理成千上万个高并发请求。

**适用场景**：高并发网络应用、高IO密集型应用（如爬虫、异步Web服务器）。

核心区别总结

| 特性         | 进程              | 线程               | 协程                  |
| ------------ | ----------------- | ------------------ | --------------------- |
| **所属层级** | 内核态 (OS)       | 内核态 (OS)        | 用户态 (用户库)       |
| **分配资源** | 独立内存空间      | 共享进程内存       | 共享线程内存          |
| **切换开销** | 极高              | 高                 | 极低                  |
| **并发能力** | 一般              | 较好               | 极高                  |
| **调度方式** | 抢占式 (操作系统) | 抢占式 (操作系统)  | 协作式 (用户代码控制) |
| **常见错误** | 内存溢出、死锁    | 线程死锁、数据竞争 | 逻辑错误、挂起不返回  |

比喻理解

- **进程**：就像是一个工厂，每个工厂有独立的工厂环境（内存）。
- **线程**：工厂里的工人，多个工人共享工厂的设备（内存），工人间工作需要协调。
- **协程**：一个工人在不同工作间快速切换（利用闲置时间），不需要管理者（OS）介入，自己决定什么时候切换，效率极高。
