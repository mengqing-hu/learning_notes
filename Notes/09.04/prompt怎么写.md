## 优秀 Prompt 怎么写

**优秀 Prompt = 角色 + 目标 + 输入 + 输出格式 + 约束 + 示例（可选）**

拆成 6 个要素：

1. **角色（Role）**：你是谁？专业度如何？
2. **目标（Goal）**：要做什么？做到什么程度？
3. **输入（Input）**：给你的材料是什么？
4. **输出格式（Format）**：JSON / 分点 / 表格 / 代码 / 段落
5. **约束（Constraints）**：禁止什么？长度？风格？
6. **示例（Few-shot）**：给 1～2 个正确样例，效果翻倍



## 写好 Prompt 的 8 条黄金原则

##### 1. 越具体，越强

不要：

> 帮我写个总结

要：

> 帮我把下面这段内容总结成 3 个要点，语言正式，不超过 100 字。

##### 2. 一次只让模型做一件事

复杂任务拆成多个 Prompt，不要堆在一起。

##### 3. 明确输出格式（超级重要）

指定：

- 分点 / 段落

- 表格

- JSON

- 代码块

- 关键词加粗

  格式越死，结果越稳。

##### 4. 加约束条件

- 长度限制
- 语言风格（口语 / 正式 / 技术 / 简洁）
- 禁止内容（不要编造、不要冗余、不要废话）
- 专业程度

##### 5. 给角色定位

> 你是资深后端工程师
>
> 你是产品经理
>
> 你是面试辅导专家
>
> 你是学术写作助手

角色越强，输出越专业。

##### 6. 用 Few-shot 示例（效果提升巨大）

给 1～2 个**正确范例**，模型会严格模仿。

##### 7. 用步骤引导（Step-by-Step）

复杂任务让模型 “一步步思考”，比直接输出强很多。

##### 8. 避免模糊词

不要：尽量、大概、差不多、简单说

要：必须、只、不超过、仅、禁止
