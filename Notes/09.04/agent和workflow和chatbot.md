# OpenClaw / Agent / Workflow 对比

## 1. OpenClaw（小龙虾）是什么？

OpenClaw 是一个**开源工具调用型 Agent 框架**，业内常戏称 “小龙虾”。

它的特点是：**让 LLM 自主判断什么时候需要调用工具、调用哪个工具**（RAG、搜索、计算、API 等），不需要提前写死流程workflow，靠模型自主决策。

一句话总结：OpenClaw 是一种**工具自主调用型 Agent**，靠模型自主决策工具使用，而不是靠固定流程。

## 2. 三者对比（直接背）

- **普通 Chatbot**

  只能问答，没有工具调用，没有决策能力，被动回复。

- **OpenClaw 这类 Agent**

  核心是**自主工具调用**：LLM 自己决定要不要查 RAG、要不要计算、要不要查外部数据，动态决策，灵活性高，但可控性弱一些。

- **Workflow**

  核心是**固定流程编排**：步骤、分支、条件都是提前定义好的，按流程一步步执行，不依赖模型自主决策，**稳定、可控、可复现**，适合企业标准化业务。

一句话总结：

- Chatbot：只会聊天
- OpenClaw Agent：会自己思考、自己调用工具
- Workflow：按固定剧本一步步执行任务
