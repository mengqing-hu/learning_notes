# 大模型应用开发落地全流程

## 1. 需求与场景定义

- 明确业务目标：问答、摘要、生成、分类、抽取、智能体、RAG 检索问答等
- 确定用户与交互方式：API 服务、小程序、Web、客服机器人、内部工具
- 约束条件：时延、并发、成本、隐私合规、是否允许外网模型

## 2. 技术方案选型

### （1）模型选择

- 通用场景：GPT-4o / Claude 3 / 文心一言 / 通义千问 / Doubao
- 私有化部署：Llama 3、Mistral、Qwen、GLM、Baichuan
- 垂直领域：法律 / 医疗 / 金融可考虑领域微调

### （2）架构路线

- 直接调用 API（最快落地）
- RAG 检索增强（解决 “知识过时、胡说、私有数据”）
- 微调 Fine-tuning / LoRA（提升格式、风格、专业度）
- Agent 智能体（工具调用、规划、多步骤任务）

## 3. 数据准备与治理

- 私有知识库清洗：PDF/Word/ 表格 / 网页 → 文本抽取、去重、分段
- 构建微调样本：指令对（instruction-output）
- 数据安全：脱敏、去隐私、合规审查

## 4. 原型开发（MVP）

- 快速搭建 Demo：LangChain / LlamaIndex / 提示词工程
- 核心流程：
  1. 用户输入
  2. 检索（RAG）
  3. 提示词构造
  4. 模型调用
  5. 结果后处理（格式化、校验、过滤）
- 验证效果：回答准确率、 hallucination 情况、时延

## 5. 提示词工程 & 效果优化

- 角色设定、约束、输出格式、示例（Few-shot）
- 加入校验逻辑：事实核查、敏感词过滤、格式校验
- 迭代优化：bad case 收集 → 修正 prompt / 补充检索库

## 6. 微调 / 垂直优化（可选）

- 样本量足够时使用 LoRA 微调
- 目标：稳定输出格式、降低幻觉、提升专业度
- 评估：ROUGE、BERTScore、人工评测、准确率

## 7. 工程化与服务部署

- 后端框架：FastAPI / Django / SpringBoot
- 向量数据库：Chroma、FAISS、Milvus、ES
- 缓存、限流、重试、日志、监控
- 部署方式：
  - 云服务（最简单）
  - 私有化 Docker + Kubernetes
- 高并发：队列、批处理、模型池

## 8. 测试与评估

- 功能测试：流程是否通
- 效果测试：准确率、召回率、幻觉率
- 压测：QPS、时延、稳定性
- 安全测试：prompt injection、敏感信息泄露

## 9. 上线与灰度发布

- 小流量灰度 → 观察日志与用户反馈
- 监控指标：调用量、失败率、时延、成本、用户满意度
- 告警机制：异常响应、超时、敏感内容

## 10. 运营与持续迭代

- 收集 bad case 入库
- 定期更新知识库
- 迭代 prompt / 微调 / 检索策略
- 成本优化：缓存、精简上下文、批量调用
