## 1. RAG 与传统微调的本质区别

RAG 是**外挂知识库**，不修改模型权重，通过外部检索引入知识，知识更新快、幻觉风险低、数据需求少；传统微调是**内化知识**，直接修改模型权重，将知识写入参数，知识更新成本高、幻觉风险高、依赖大量标注数据，二者核心是知识注入方式的本质差异。



## 2. 为什么 RAG 能缓解大模型的幻觉问题？

大模型幻觉源于训练数据截止、参数记忆偏差、生成无事实约束。RAG 通过**事实锚定**（强制基于检索的真实文档生成）、**知识补全**（补充实时 / 私有知识）、**可追溯校验**（提供回答来源），从根源上减少编造，缓解幻觉。



## 3. 如何定义 RAG 中的 “相关文档”？

相关文档是指在给定用户查询（Query）下，从知识库中通过 语义相似度（如向量检索）或关键词匹配（如 BM25）筛选出的 Top-K 最相关文本片段。其 “相关性” 通常由检索模型打分决定，核心要求是能为用户问题提供有效事实支撑、语义与意图高度匹配。



## 4. RAG 中主流检索器有哪些？各自优势是什么？

- **稀疏检索（如 BM25）**：基于词频和逆文档频率，擅长精确关键词匹配，计算高效，可解释性强，但缺乏语义泛化能力，无法处理同义词、多义词。
- **稠密检索（如 DPR、Sentence-BERT）**：将文本映射为稠密向量，支持语义层面的相似性搜索，泛化性好，能匹配上下文相关内容，但依赖高质量嵌入模型和算力。
- **混合检索**：结合稀疏与稠密方法（如 RRF 融合），兼顾关键词精准性和语义泛化能力，同时提升整体召回率与准确率，是工业界主流方案。



## 5. 知识库文档分块有哪些常用策略？

- **固定大小分块**：按固定 Token / 字符数分块，可设置重叠窗口，实现简单、效率高，但易破坏语义完整性。
- **语义分块**：通过 Embedding 计算语义相似度，将语义连贯内容合并为块，保证块内语义完整，但计算成本高。
- **结构感知分块**：按文档天然结构（标题、章节、段落）分块，保留层级关系，适合结构化长文档，但依赖文档格式。
- **递归分块**：按层级递归拆分（章节→段落→句子），兼顾结构和语义，是 LangChain 等框架的默认方案。
- **大模型分块**：用大模型识别语义单元自动分块，分块质量高，但成本高、延迟高，适合小规模知识库。



## 6. 为什么需要对向量进行归一化？

向量归一化（通常为 L2 归一化）可**消除向量模长对相似度计算的影响**，让余弦相似度等价于向量点积，大幅提升向量库检索效率；同时仅保留语义方向信息，避免模长差异导致的相似度偏差，统一度量标准，提升检索稳定性。



## 7. 生成阶段如何融合检索结果？

- **基础拼接**：按相似度从高到低直接拼接，实现简单，适合小规模结果，但易冗余、无逻辑。
- **重排序后拼接**：用 CrossEncoder 等模型二次排序后拼接，提升高相关片段权重，减少干扰。
- **摘要融合**：先对检索片段做摘要提取核心信息，再拼接输入，压缩上下文长度，适配长上下文限制。
- **大模型融合**：用大模型对检索结果做去重、冲突消解、信息整合，解决多文档复杂场景，是工业界最优方案。
- **结构化融合**：按主题、时间等维度分类，结构化插入 Prompt，逻辑清晰，适合多维度查询。



## 8. 如何避免 RAG 生成无关或错误内容？

从全链路优化：检索环节优化分块、采用混合检索 + 重排序、设置相似度阈值、维护高质量知识库；融合环节做去重、冲突消解、限制上下文长度；生成环节设计强约束 Prompt、强制标注来源、增加生成后事实校验，拦截幻觉内容。



## 9. 为什么需要重排序（Re-ranking）？常见方法有哪些？

初始检索是粗筛，仅基于向量相似度排序，存在语义偏差、噪声干扰等问题；重排序是精排，通过更精准的模型二次排序，提升高相关文档排名、过滤无关内容，大幅提升回答质量。

常见方法：CrossEncoder（精度高、成本高）、ColBERT（兼顾精度与效率）、LLM-based 重排序（适配复杂场景）、RRF（融合多检索器结果，无需训练）、传统机器学习方法（可解释性强）。



## 10. RAG 与微调如何结合？

二者为互补关系：一是用微调优化 RAG 核心组件，如微调 Embedding 模型提升检索精度、微调重排序模型、微调大模型适配 RAG 场景；二是用 RAG 增强微调效果，为微调提供高质量实时数据，推理阶段补充知识；三是端到端结合，如 Agent RAG、Self-RAG、检索增强微调。



## 11. 如何评估 RAG 生成结果的质量？

从三个维度评估：

- **检索质量**：召回率、精确率、F1 值、MRR、NDCG，衡量检索的全面性、准确性与排序质量。
- **生成质量**：人工评估（准确性、完整性、连贯性、有用性）、自动评估（事实一致性、BLEU/ROUGE/BLEURT）、幻觉率统计。
- **用户体验**：满意度调研、任务完成率、响应延迟，衡量系统实用性与性能。



## 12. 如何优化检索的召回率（Recall）？

通过优化分块（更小块、更大重叠）、查询扩展（同义词、问题重写）、混合检索（融合稀疏 + 稠密）、降低相似度阈值、多轮迭代检索、微调领域 Embedding 模型、完善知识库索引等方式，提升检索覆盖度，避免漏检相关文档。



## 13. RAG 如何处理多文档中的冲突信息？

核心流程为**识别冲突→评估可信度→融合 / 标注→生成回答**：先通过大模型识别冲突点，再按来源权威性、时效性、一致性评估文档可信度，优先采用高可信度内容，或标注冲突、中立总结，同时在 Prompt 中明确约束，禁止编造结论。



## 14. 如何解决 “检索偏好” 问题？

检索偏好指检索器对特定类型文档存在偏好，导致结果失衡。解决方案：用重排序修正偏好、分类型检索融合、查询去偏、Embedding 模型去偏、增加结果多样性约束、多检索器融合中和偏好、定期人工校准优化策略。



## 15. 如何优化长文档的检索效果？

针对长文档分块破坏语义、上下文丢失的痛点，采用结构感知分块、层次化检索（先粗筛章节再细粒度检索）、长上下文 Embedding、摘要增强检索、多模态检索、迭代检索、大模型分块等方案，保留完整上下文，提升检索精度。



## 16. HyDE 的原理是什么？

HyDE（假设文档嵌入）是查询优化技术，核心原理：用户查询是 “问题”，知识库文档是 “陈述性文本”，二者语义分布存在差异；用大模型根据查询生成符合知识库风格的假设回答文档，再用该假设文档做 Embedding 检索真实文档，弥合语义鸿沟，提升检索匹配度。



## 17. 什么是迭代检索（Iterative Retrieval）？

迭代检索是多轮检索优化技术，核心逻辑：第一轮用原始查询检索生成初步回答，再用大模型识别信息缺口、生成补充查询，多轮循环检索补充上下文，直到信息足够生成最终回答，解决单轮检索信息不足问题，适配复杂多维度查询。



## 18. Self-RAG 的核心创新点是什么？

Self-RAG 的核心是让大模型**全流程自主决策与优化**：自主判断是否需要检索、自主评估检索结果的相关性与可信度、自主在生成过程中反思修正，无需人工干预，从根源上减少幻觉，大幅提升 RAG 的鲁棒性、准确性与效率。



## 19. RAG 如何适配实时更新的知识库？

通过增量式向量库更新（仅更新变更文档，无需全量重建）、实时数据同步（对接 CDC、消息队列）、冷热数据分离、高频查询缓存、Agent 实时 API 兜底、定期全量校验等方案，实现低延迟、高可用的知识库实时更新，保证检索时效性。



## 20. 用户问 “2025 年诺贝尔奖获得者”，但知识库中无该信息，RAG 如何处理？

检索后无相关内容时，通过强约束 Prompt 明确告知大模型不得编造，生成规范回答：明确说明知识库未收录相关信息，无法提供准确回答，可补充引导用户通过官方渠道查询；进阶方案可触发实时搜索引擎兜底，同时记录查询提醒管理员更新知识库。
