## 为什么同一个问题多次询问会得到不同答案?

LLM的本质是预测下一个Token，每次返回的是一个概率分布，所以一次回答是多次概率采样的连锁结果。

如果模型每次都只选概率最高的那一个（贪婪搜索），那么它确实会永远给出相同的答案。但在实际应用中，为了让语言更自然、更有创造力，会引入**随机采样**。

需要注意的是这里的“不同”并不等同于“错误”。在向量空间中，“正确答案”往往不是唯一的，尤其是解释类、总结类、推理路径类的问题。LLM的目标不是“一致性”，而是在统计意义上像人类写的文本.



## 为什么 LLM 被认为“天生不稳定”?

#### 1. **模型是概率性的，而非确定性的**

- 大模型本质上是一个**基于概率的生成系统**。它通过预测下一个词的概率分布来生成文本。
- 即使输入完全相同，采样过程（如 `top-k`、`temperature` 等）也会引入随机性。
- 例如：使用高温度（temperature）时，输出更随机；低温度时更保守，但仍可能因采样差异导致不同结果。

结论：**不是模型“出错”，而是其设计本就是随机的**。这并非缺陷，而是其表达多样性和创造力的基础。

#### 2. **训练数据的复杂性与噪声**

- LLM 训练数据来自互联网，包含大量不一致、矛盾甚至错误的信息。
- 模型学习的是“统计规律”，而不是“真理”。因此它可能“合理地”生成一个听起来正确但事实上错误的内容（即幻觉）。
- 例如：模型可能根据“某人出生于1980年”和“他活了150岁”等信息，推断出一个虚构的生平。

结论：模型没有“真实感”，它只是模仿语言模式，无法验证事实真伪。

#### 3. **上下文依赖性强，微小变化引发巨大差异**

- 模型对输入提示（prompt）极其敏感。哪怕只改一个词，也可能导致输出完全不同。
- 例如：
  - “请解释量子力学” → 详细科普
  - “请用小学生能懂的方式解释量子力学” → 完全不同的风格和深度

原因：模型是“上下文感知”的，它的响应高度依赖于提示的措辞、语气、结构。

#### 4. **缺乏真正的推理与常识**

- 模型没有意识、没有世界知识库，它只是在“拟合”语言模式。
- 当遇到超出训练数据范围的问题（如新概念、极端情况），模型容易“胡说八道”。
- 它不会“思考”，而是“回忆”并重组已有语料。

举例：问“如果猫会飞，它会怎么飞？”——模型可能给出看似合理的描述，但这是虚构逻辑。

#### 5. **训练目标与人类期望存在偏差**

- 模型的目标是“最大化下一个词的预测准确性”，而不是“生成真实、一致、安全的内容”。
- 因此，它可能在“流畅度”上表现优秀，但在“真实性”、“一致性”、“安全性”上表现不佳。

## 是否“天生”就不稳定？

关键点：**这不是“天生”的缺陷，而是“设计使然”**。

- 如果我们要求模型“绝对稳定”、“永远正确”、“无幻觉”，那它就不再是语言模型，而是一个逻辑推理引擎或知识数据库。
- 但现实需求是：我们需要一个能灵活应对各种任务、生成自然语言、具备创造性的系统



## 如何缓解“不稳定性”？

虽然不能彻底消除，但可通过以下方式降低风险：

| 方法                               | 说明                                               |
| ---------------------------------- | -------------------------------------------------- |
| **提示工程（Prompt Engineering）** | 优化输入提示，明确指令，减少歧义                   |
| **温度控制 & 采样策略**            | 降低 temperature，使用 beam search，提高输出一致性 |
| **后处理与过滤**                   | 使用规则、关键词检查、事实核查工具                 |
| **检索增强生成（RAG）**            | 引入外部知识源，减少幻觉                           |
| **模型微调（Fine-tuning）**        | 针对特定任务调整模型行为                           |
| **多轮对话与上下文管理**           | 保持一致性，避免前后矛盾                           |



## LLM 擅长什么，不擅长什么?

#### LLM 特别擅长的事

##### 1. 自然语言理解与生成

- 读懂、总结、改写、扩写、摘要、对话、写作
- 模仿风格、语气、逻辑结构
- 例子：写邮件、写代码注释、润色论文、对话聊天

##### 2. 知识问答与信息检索（基于训练语料）

- 常识、概念、原理、历史、科普
- 给框架、给思路、给步骤
- 例子：解释 Transformer、讲算法思路、给方案

##### 3. 逻辑推理（浅层）

- 类比、归纳、常识推理
- 根据上下文补全逻辑
- 例子：A 是 B 的爸爸，B 是 C 的爸爸 → A 是 C 的爷爷

##### 4. 代码能力（很强）

- 写代码、改 Bug、解释代码、写注释
- 常见算法、数据结构、工具使用
- 例子：写快排、写 SQL、写 Python 脚本、写 Dockerfile

##### 5. 结构化内容生成

- 表格、清单、大纲、思维导图、方案
- 按格式输出：JSON、Markdown、CSV
- 例子：面试提纲、项目方案、周报

##### 6. 多轮对话与上下文理解

- 记住前面几轮内容
- 连贯对话、持续修改
- 例子：AI 助手、客服、陪练

#### LLM **非常不擅长**的事

##### 1. 精确数值计算 / 复杂数学

- 大数加减乘除、复杂公式、高精度计算
- 容易算错、逻辑对结果错
- 例子：123456789 × 987654321，LLM 经常算错

##### 2. 实时信息 / 最新数据

- 训练数据截止到某个时间点
- 不知道之后发生的新闻、股价、政策、版本更新
- 例子：今天天气、最新比赛结果、刚发布的模型

##### 3. 长期记忆与超大文本精读

- 上下文窗口有限
- 超长文档容易 “看前面忘后面”
- 例子：100 页 PDF 精细抽取、跨章节推理

##### 4. 真正 “理解” 世界（没有实体概念）

- 没有视觉、听觉、触觉
- 不知道苹果是什么味道、红色长什么样
- 只是文字概率模型

##### 5. 严谨事实保证（容易幻觉）

- 会编造：人名、地名、数据、论文、引用、代码
- 听起来很真，实际是瞎编
- 例子：假文献、假 API、假参数

##### 6. 复杂逻辑与形式化推理

- 数学证明、复杂逻辑题、多步推导
- 一步错步步错
- 例子：几何证明、数论问题、复杂逻辑谜题

##### 7. 自我纠错能力弱

- 自己很难发现自己错了
- 你指出错误，它才可能改
- 不具备真正的反思与验证

##### 8. 价值观 / 道德 / 伦理不稳定

- 容易被诱导输出不当内容
- 需要对齐、RLHF、安全机制

##### 9. 真正 “学习” 新知识

- 推理时不能更新权重
- 只能靠上下文学习（few-shot/zero-shot）
- 不能像人一样 “学会并记住”
