# 向量数据库常见索引类型

## 1. **IVF（倒排文件索引）**

- 原理：**先聚类，再分桶**
- 过程：把向量聚成很多簇，查询只搜最近的几个簇
- 优点：**数据量大也能用，省内存**
- 适合：百万～千万级向量
- 例子：像图书馆先按分类放书，只查一个分类

## 2. **PQ（乘积量化）**

- 原理：**把向量切分段，每段用编号代替**
- 作用：**压缩向量，大幅省空间**
- 优点：存储极省，适合海量数据
- 缺点：会损失一点精度
- 例子：把长向量 “压缩编码”，像图片压缩

## 3. **IVF_PQ（最常用组合）**

- 原理：**IVF 分桶 + PQ 压缩**
- 优点：**速度、内存、精度三者平衡**
- 场景：Milvus、FAISS 里默认常用索引
- 面试必说：**工业界最主流**

## 4. **HNSW（层次化导航小世界）**

- 原理：**构建多层邻居图**
- 过程：向量之间连边，查询一层层跳向最近点
- 优点：**查询极快，毫秒级**
- 缺点：**占内存大**
- 适合：对延迟要求高的场景（搜索、推荐）

## 5. **FLAT（暴力索引 / 线性扫描）**

- 原理：不建索引，**挨个算距离**
- 优点：**最精准，100% 召回**
- 缺点：数据一多就巨慢
- 适合：**小数据量、做基准测试**





# 例子

## 例子 1：图书馆找书（对应 IVF 索引）

#### 没索引时

图书馆有 100000 本书，你要找 **“机器学习”**。

你必须：

一本一本翻标题 → 超级慢。

#### 建索引的过程

1. **先分类（聚类）**

   管理员把书分成很多类：

   - 计算机类

   - 文学类

   - 医学类

   - 历史类

     这一步就是 训练索引。

2. **给每本书贴标签，放进对应类别**

   《机器学习》→ 计算机类

   《红楼梦》→ 文学类

   这一步就是 **建立倒排索引**。

3. **你现在搜索**

   搜 “机器学习”。

   系统一看：属于**计算机类**。

   只在计算机类里找，**不用遍历全馆**。

## 例子 2：朋友圈找相似照片（对应 HNSW 索引）

你有 10000 张照片，每张照片转成一个向量。

#### 没索引

找 “和这张猫最像的图”，要对比 10000 次。

#### 建索引的过程

1. **给每张图找 “最像的邻居”**

   - 猫图 → 只连别的猫图

   - 风景图 → 只连别的风景图

     形成一张图结构。

2. **搜索时**

   从某张图开始，**一跳一跳找邻居**，

   不用看全量图片，**速度极快**。



## 不同数据库的索引策略

| 数据库   | 是否需要手动设置 | 使用索引        |
| -------- | ---------------- | --------------- |
| Chroma   | ❌ 不需要         | HNSW            |
| Weaviate | ⚠️ 调参数         | HNSW            |
| Qdrant   | ⚠️ 调参数         | HNSW            |
| Milvus   | ✅ 需要           | IVF / HNSW / PQ |
| FAISS    | ✅ 完全手动       | 全部支持        |
| Pinecone | ❌ 不可见         | 类 HNSW         |
