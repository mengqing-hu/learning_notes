# CDN加速

CDN（Content Delivery Network，内容分发网络）是一套**分布式服务器网络**，核心是把网站 / 应用的静态内容（图片、视频、CSS/JS 等）缓存到全球各地的**边缘节点**，让用户从最近节点获取内容，大幅降低延迟、提升加载速度并减轻源站压力。

### 一、核心原理

用户请求 → 智能 DNS 选最近节点 → 节点缓存命中则直接返回；未命中则回源拉取并缓存，后续请求复用缓存。

### 二、工作流程

1. **用户发起访问**：浏览器输入网址，请求资源（如图片、视频）。
2. **智能调度**：CDN 的全局负载均衡（GSLB）根据用户 IP、运营商、节点负载，分配**最近 / 最优边缘节点**。
3. **缓存检查**：
   - **缓存命中**：节点直接返回内容，延迟通常几十毫秒。
   - **缓存未命中**：节点向源服务器拉取内容，缓存后再返回用户。
4. **内容更新**：通过缓存过期、主动刷新 / 预热，保证内容时效性。

### 三、核心优势

- **极致加速**：静态资源就近分发，跨国 / 跨区域延迟降低 **50%–80%**，页面加载速度显著提升。
- **减负源站**：70%+ 流量由边缘节点承载，源站带宽、计算压力大幅下降，避免高并发崩溃。
- **高可用**：多节点冗余 + 自动故障切换，部分节点故障不影响整体服务。
- **安全增强**：内置 DDoS 防护、WAF，拦截恶意攻击，保护源站安全。
- **降本增效**：减少源站带宽与服务器成本，综合运营成本可降 **30%–50%**。

### 四、主要加速类型

- **静态加速（基础）**：缓存图片、CSS/JS、HTML、视频等不变内容，最常用。
- **动态加速**：优化 API、登录、搜索等动态请求的传输路径，选最优路由、优化 TCP/QUIC 协议。
- **全站加速**：动静内容智能拆分，静态缓存 + 动态优化，一站式解决全站性能问题。
- **流媒体加速**：视频切片缓存、热点分发，支持直播 / 点播低延迟、高并发播放。

### 五、典型应用场景

- 电商网站：商品图片、详情页、静态资源加速，提升转化率。
- 视频 / 直播平台：低延迟、高并发分发，保障流畅观看。
- 游戏：更新包、资源包快速下载，降低登录 / 加载延迟。
- 企业官网 / SaaS：全球用户访问一致的高速体验。
- 大文件下载：软件安装包、安装镜像等，提升下载速度与稳定性。

### 六、与传统访问的区别

|  对比项  |         传统直连源站          |           CDN 加速            |
| :------: | :---------------------------: | :---------------------------: |
| 数据路径 | 用户 → 源站（跨地域、长链路） | 用户 → 最近边缘节点（短链路） |
|   延迟   |     高（跨洋可达 200ms+）     |    低（本地节点 30ms 内）     |
| 源站压力 |     全量请求直连，易过载      |    仅首次 / 未缓存请求回源    |
|  可用性  |      单点故障即服务中断       |     多节点冗余，自动切换      |
|  安全性  |    源站直接暴露，易受攻击     |    边缘节点防护，源站隐藏     |

### 七、总结

CDN 就是互联网的 **“分布式快递网”**：把内容提前 “存” 到用户家门口，让访问更快、源站更稳、成本更低、体验更好。
