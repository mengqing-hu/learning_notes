# 1. Workflow（工作流）

**解释**：把一个复杂任务拆成**固定、有序、可编排**的步骤，按顺序或条件一步步执行，整个流程预先定义好，执行时不自主决策，强调**稳定、可控、可复现**。

**核心特点**：==步骤固定、顺序固定、逻辑固定、无自主思考。==

**应用场景**：

- 标准化数据处理：清洗 → 提取 → 格式转换 → 入库
- 审批流程：提交 → 初审 → 复审 → 通过 / 驳回
- 固定 AI 问答流程：用户提问 → 检索知识库 → 生成回答 → 合规检查 → 输出
- 报表生成：拉取数据 → 统计 → 绘图 → 发送邮件

# 2. Tool / Function Calling（工具 / 函数调用）

**解释**：大模型以 结构化格式（通常是 JSON）明确声明 “要调用哪个工具、传入什么参数”，由系统执行工具并返回结果，让模型具备使用外部能力的能力。

**核心特点**：模型只负责 “决定调用什么 + 参数”，不直接执行，执行由外部系统完成。

**应用场景**：

- 查询类：查天气、查订单、查物流、查数据库
- 操作类：发邮件、发短信、创建日程、修改文档
- 计算类：复杂计算、代码执行、公式运算
- 接口类：调用支付、物流、地图、风控等第三方 API

#### 2.1 Function Calling

- 最早由 OpenAI 提出

- 强调：==调用一个**函数**==，传参数，拿返回值

- 格式典型是：

  ```
  {"name":"get_weather","parameters":{"city":"北京"}}
  ```

#### 2.2Tool Calling

- 后来扩展后的统称
- 范围更大：==不仅是函数，**还可以是工具、插件、技能、API、数据库查询等**==
- 现在主流框架（LangChain、LlamaIndex、豆包等）都叫 **Tool Calling**
