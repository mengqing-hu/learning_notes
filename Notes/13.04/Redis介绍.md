# Redis 是什么？

**Redis = Remote Dictionary Server**

是一款 **基于内存、高性能、键值对（Key-Value）**的 **NoSQL** 数据库，**最常用作缓存**，也能做消息队列、分布式锁、计数器等。

### 核心特点

1. **纯内存为主，速度极快**

   - 读写在内存，**单机 10w+ QPS** 很轻松
   - 可持久化到磁盘，防止重启丢失

2. **数据结构超级丰富**

   不只是简单 K-V，支持：

   - String（字符串）
   - Hash（哈希）
   - List（列表）
   - Set（集合）
   - ZSet（有序集合）
   - Bitmap、HyperLogLog、GEO、Stream 等

3. **单线程 + IO 多路复用**

   - 简单稳定
   - 无锁竞争，并发高

4. **功能极强**

   - 缓存
   - 分布式锁
   - 限流
   - 消息队列
   - 排行榜
   - 签到、点赞、关注关系
   - 会话存储

### 最典型使用场景

1. **热点数据缓存**

   商品详情、用户信息、首页配置、接口结果

   → 扛高并发、减轻 DB 压力

2. **分布式锁**

   秒杀、库存扣减、防止重复提交

3. **计数器 & 限流**

   接口限流、点赞数、阅读数、UV/PV

4. **排行榜**

   游戏积分、热度排行（用 ZSet）

5. **消息队列**

   简单任务队列、延迟队列

6. **Session 共享**

   多台服务器登录态统一

### 为什么大家都用 Redis？

- 性能吊打传统数据库
- 用法简单，API 友好
- 生态成熟，几乎所有语言都有客户端
- 功能全能，一个组件顶好几个用

### Redis 缓存三大问题

##### 1. 缓存穿透

###### 是什么

查询**根本不存在的数据**。

缓存里没有 → 去查数据库 → 数据库也没有 → 每次都穿透到 DB。

恶意刷请求就能把数据库打挂。

例：查商品 ID = -1、999999999

###### 解决方案

- **缓存空值**

  查不到也存 `null`

   或空对象，设置短暂过期（30s～1min）

- **布隆过滤器（Bloom Filter）**

  提前把存在的 ID 塞进布隆过滤器，不存在直接拒绝

- 参数校验、黑名单、接口限流

##### 2. 缓存击穿

###### 是什么

**某个热点 Key 过期了**，同一瞬间大量请求同时进来，全部打到数据库。

只有一个 Key 出事，叫 “击穿”。

例：爆款商品缓存过期，秒杀开始瞬间。

###### 解决方案

- **热点 Key 永不过期**

- **加互斥锁（分布式锁）**

  只让一个线程去查库重建缓存

- 后台定时主动刷新缓存

##### 3. 缓存雪崩

###### 是什么

**大量 Key 同一时间集体过期** 或 **Redis 宕机**，所有请求同时压到数据库 → DB 直接崩。

###### 解决方案

- **过期时间加随机值**，避免集体过期
- **Redis 高可用（主从 + 哨兵 / 集群）**
- **多级缓存**（本地缓存 + Redis）
- **限流 + 降级**
- 缓存预热

### Redis 高可用：主从 / 哨兵 / 集群

##### 1. 主从复制（Master-Slave）

- **主节点写**，**从节点读**
- 分担读流量
- 主挂了，从不能自动上位

作用：**读写分离、备份**

##### 2. 哨兵（Sentinel）

- 监控主节点状态
- 主挂了 **自动故障转移**，从节点升主
- 自动切换，无需人工干预

作用：**高可用 + 自动容灾**

##### 3. 集群（Redis Cluster）

- **数据分片存储**（16384 个槽）
- 多主多从，可横向扩展
- 自带高可用，自动迁移槽位

作用：**海量数据 + 高并发 + 横向扩容**
